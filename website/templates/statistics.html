P-values and significance
  • p < 0.05 — statistically significant at the conventional α = 0.05 threshold.
  • p < 0.01 — highly significant;  p < 0.001 — very highly significant.
  • A significant p-value does not tell you the size or biological importance of the effect — always inspect the actual data and effect size.
Box plots
  • Box = interquartile range (IQR; 25th–75th percentile). Line inside box = median.
  • Whiskers extend to 1.5 × IQR beyond the box edges. Dots outside whiskers = potential outliers.
  • Individual data points (swarm) show the actual measurements; if N is small, inspect them directly rather than relying only on the box.
Choosing a significance test
  • ANOVA — compares means of 3+ groups; assumes normality (Shapiro–Wilk p > 0.05) and equal variances (Levene p > 0.05).
  • Kruskal–Wallis — non-parametric alternative to ANOVA; use when normality or variance assumptions are violated.
  • Tukey HSD — post-hoc test after ANOVA; shows which specific group pairs differ. Significance letters group means that are not significantly different from each other (same letter = no significant difference).
Assumptions tests
  • Shapiro–Wilk: p > 0.05 = no evidence against normality — the parametric ANOVA is appropriate.
  • Levene: p > 0.05 = variances are homogeneous across groups — the equal-variance ANOVA assumption is met.
  • With small samples (N < 5 per group), these tests have low power — consider using non-parametric tests regardless.

About this table & how to read results

About this table: The summary below shows the combined results of two statistical assumption tests run for each variable:

  • Shapiro-Wilk (Normality): tests whether the data within each group follows a normal distribution. A p > 0.05 indicates normality is met.
  • Levene's Test (Homogeneity of Variances): tests whether the spread of values is similar across all groups. A p > 0.05 indicates equal variances.

Both tests are evaluated together to decide the most appropriate significance test (ANOVA, Welch's ANOVA, or Kruskal-Wallis).

Explore further in the other sub-tabs:

  • Box Plots: inspect the raw distribution and spread of individual data points for each variant and variable.
  • Residuals vs Fitted: visually check for homogeneity of variances — points should be evenly scattered around the horizontal axis with no pattern.
  • Normal Q-Q: visually assess normality — points following the diagonal reference line indicate normally distributed residuals.
How to read these plots & colour coding

What these plots show: One box plot per variant (group) for each variable, showing the full data distribution. Use these to visually inspect whether each group is roughly symmetric and whether spread (variance) differs markedly between groups — both key ANOVA assumptions.

  • Green — group passed the Shapiro-Wilk normality test (p > 0.05).
  • Red — group failed the Shapiro-Wilk normality test (p ≤ 0.05).

Color reflects only the per-group Shapiro-Wilk result. Levene's test is across all groups simultaneously and is shown in the Assumptions Summary table only.

Outlier removal: Points beyond 1.5× IQR are flagged as red dots. Click any red dot to permanently exclude it and recalculate. Remove outliers before running ANOVA tests.

Residuals vs Fitted — how to interpret & outlier removal

Residuals vs Fitted — Primary purpose: Homogeneity of Variances

Displays residuals (observed − group mean) on the Y-axis against fitted values (group means) on the X-axis. Because each group occupies one X position, the vertical spread at each position directly shows within-group variance — making it easy to spot whether spread is consistent across groups (Levene's assumption).

Well-behaved data: Points scattered evenly around zero with similar vertical spread across all groups.
Warning signs: Funnel/fan shape → unequal variances (heteroscedasticity); Curved trend → non-linearity or non-normality; Isolated points far from zero → potential outliers.

Outlier identification (|z| > 2): Raw residuals are standardized (÷ SD) to a dimensionless z-score. Points with |z| > 2 — beyond the ~95% range of a normal distribution — are flagged in red. Before removing, ask: measurement error? → remove; real biological extreme? → keep; many points flagged on one side? → systematic variance issue, removal won't help; does Levene's p change after removal? → re-run to verify.

Click any red point to exclude it permanently and recalculate. Blue points are not clickable.

Normal Q-Q — how to interpret & outlier removal

Normal Q-Q — Primary purpose: Normality of Residuals

Compares the observed quantiles of the residuals (Y-axis) against the theoretical quantiles of a perfect normal distribution (X-axis). If the data is truly normal, both sets of quantiles match and points lie along the diagonal reference line. Deviations reveal exactly where and how the distribution differs from normality.

Well-behaved data: Points lie closely along the diagonal; small deviations at the ends are acceptable.
Warning signs: S-shaped curve → heavy/light tails; one-end bend → skewness; gaps or jumps → bimodality; extreme off-line points → outliers.

Outlier identification (|std residual| > 2): The Y-axis already shows standardized residuals, so the threshold is readable directly from the plot. Points with |std residual| > 2 — beyond the ~95% range expected for normal data — are flagged in red; they also tend to deviate most from the diagonal line. Before removing, ask: measurement error? → remove; real biological extreme? → keep; consistent pattern across many points (S-curve, bend)? → distribution is globally non-normal, removal won't help — consider data transformation; does Shapiro-Wilk p pass after removal? → re-run to verify.

Click any red point to exclude it permanently and recalculate. Blue points are not clickable.

3+ Factors

Assumptions fail → ART ANOVA + per-variable follow-up (Bonferroni)

Post-hoc comparisons shown only when overall test is significant (p < 0.05). All p-values corrected for multiple comparisons.

Continuous factors not supported. This tool treats all factors as categorical. Dose-response curves, time-series, or any factor with a meaningful numeric ordering (e.g. 0 µM, 10 µM, 100 µM dose) should be analysed with regression or repeated-measures ANOVA in dedicated software (R, SPSS, Prism). Using such data here may produce misleading letter groups.
Hierarchical / nested designs not supported. The tool assumes a fully crossed, independent-groups design. Split-plot, repeated measures, or nested experiments (e.g. multiple technical replicates per biological replicate) require mixed-effects models and are not handled here.

How PCA works in this tool
What PCA does

PCA finds new axes (principal components) that capture the largest sources of variation in your data. Each sample gets a score on these axes; the biplot shows how samples cluster and which variables drive the separation.

All variables are standardised (mean = 0, unit variance) before analysis, so variables on different scales contribute equally. Wide datasets (more variables than samples) are handled correctly — at least 2 samples are required after filtering.

Reading the biplot
  • Points — individual samples. Closer = more similar across all selected variables. Horizontal = PC1 score, vertical = PC2.
  • Ellipses — 95% confidence region per group (2 SD of bivariate PC distribution). Overlapping = groups not clearly separated.
  • Arrows (loadings) — direction shows which samples have high values; length shows contribution strength. Similar directions = positive correlation; opposite = negative.
  • Axis labels (%) — fraction of total variance captured by that component.
Missing data strategies

SVD-based PCA cannot run with missing values. Five strategies are available:

  • Hybrid (recommended) — variables exceeding the threshold % missing are excluded; remaining gaps filled with group/column mean. No samples lost.
  • Drop rows — removes any sample with at least one missing value. Simple but can lose many samples.
  • Impute group/column mean — all missing values replaced with group or column mean. All data kept, but imputed values compress within-group variation.
  • Exclude variables — drops variables exceeding the threshold; remaining rows with missing values are then dropped.
  • KNN — estimates missing values from the k most similar rows. More sophisticated, less interpretable on small datasets.
PCA algorithms
  • Standard SVD — fast, numerically stable, requires complete data. Apply a missing-data strategy first. Best when missingness is low.
  • NIPALS — uses only observed cells per iteration, no imputation. Standard in metabolomics/proteomics for structured missingness (different variables measured on different sample subsets).
  • EM-PCA — iteratively refines missing values via low-rank reconstruction. Preserves covariance structure better than mean imputation; faster than NIPALS on large datasets.
Which algorithm to choose?
  • Complete data or few scattered missing values → SVD + Hybrid or KNN
  • Different variables measured on different sample subsets (block missingness) → NIPALS
  • Many scattered missing values across a large dataset → EM-PCA
  • Compare conditions rather than individual replicates → enable Average replicates by factor(s) with any algorithm
Loading arrows & controls

Arrow length = Euclidean magnitude in the current 2D plane. Long = well-represented by visible PCs; short = most variance captured by other PCs.

Controls (when "Show variable loadings" is on): Arrow scale — stretch/shrink all arrows visually. Min. magnitude — hide arrows below threshold. Top N — show only the N longest arrows. Both filters also highlight matching rows in the loadings table below the plot.

Kaiser criterion:

The Kaiser criterion (eigenvalue > 1 rule) retains only components that explain more variance than a single standardised variable would on its own (i.e. > 1/nvars of total variance).

How to use it: If k components are suggested, PC1–PCk are the axes most worth examining. Components beyond k mostly capture noise. That said, treat this as a starting point — inspect the scree plot and cumulative explained variance as well. For exploratory analysis, PC1 vs PC2 (the default view) is usually the right place to start.

Caveat: The criterion can over-retain components when variables are many and weakly correlated, and under-retain when they are few and highly correlated. "Retain 0 components" means no component clears the per-variable threshold — this is common with large, loosely-related variable sets and does not indicate an analysis error.

How OPLS and OPLS-DA work in this tool
What OPLS does

OPLS (Orthogonal PLS) is a supervised method that decomposes X into two parts: predictive variation (correlated with Y) and orthogonal variation (unrelated to Y — structured noise). By removing the orthogonal part, the predictive component becomes cleaner and easier to interpret than standard PLS.

OPLS-DA (Discriminant Analysis) uses a binary or multi-class group membership vector as Y. It is the most common use case — finding the directions in X that best discriminate between defined groups.

Reading the plots
  • Score plot (tp vs to): samples on the predictive component (x) and first orthogonal component (y). Group separation appears along x; vertical spread is structured within-group variation unrelated to group membership.
  • S-plot: each variable plotted as covariance with tp (x) against correlation with tp (y). Top-right corner = high contribution and high reliability. Variables near origin contribute little.
  • VIP plot: Variable Importance in Projection — summarises each variable's contribution across all components. VIP > 1 is the conventional threshold for importance in discrimination.
Parameters
  • Orthogonal components: number of orthogonal (noise) components to remove. Start with 1; increase only if R²X rises substantially without improving Q².
  • Response variable (Y): for OPLS-DA select a categorical column (group labels); for OPLS select a continuous outcome.
  • Q² (cross-validation): leave-one-out CV. Q² > 0.5 indicates a useful predictive model. Large gap between R²Y and Q² suggests overfitting.
Model metrics
  • R²X: fraction of total X variance explained by the model.
  • R²Y: fraction of Y variance explained (goodness of fit).
  • Q²: cross-validated predictive ability — the most important metric for assessing whether the model is genuinely predictive rather than just fitting noise.

How PLS and PLS-DA work in this tool
What PLS does

PLS (Partial Least Squares) finds a sequence of latent variables (LVs) that simultaneously decompose X and Y while maximising the covariance between them. Unlike OPLS, PLS does not separate orthogonal variation — all components contain a mixture of predictive and non-predictive variation.

PLS-DA (Discriminant Analysis) uses binary-encoded class membership as Y. It is directly comparable to OPLS-DA and often gives similar results. PLS-DA can be used with any categorical column.

Reading the plots
  • Score plot (T1 vs T2): samples projected onto the first two LV scores. Group separation along T1 indicates LV1 captures between-group variation. Axis labels show per-component R²X contributions.
  • Weights plot (W1 vs W2): each variable as an arrow to its weight coordinates. Variables pointing in similar directions are correlated; large distances from origin = high importance. Arrow colour: red ≥ 1.5 VIP, orange ≥ 1.0, grey < 1.0.
  • VIP plot: Variable Importance in Projection — combines all LV components. VIP > 1 is the conventional threshold for importance.
Model metrics
  • R²X: cumulative X variance explained by retained LVs.
  • R²Y: Y variance explained (goodness of fit).
  • Q²: cross-validated predictive ability. Values > 0.5 indicate a useful model. A large R²Y–Q² gap suggests overfitting.

Initial Data Cleaning & Pre-selection
Samples (rows) missing more than (100 − threshold)% of the selected variables are excluded before analysis. Raise the threshold to keep only well-measured samples; lower it to retain more samples at the cost of sparser data.
0%
Variables that barely change across samples carry no useful information — they inflate correlations and distort PCA axes. The Coefficient of Variation (CV = SD ÷ mean × 100%) quantifies relative spread: a variable with CV = 3% changes by only 3% of its mean across all samples. Any variable whose CV falls below the threshold is excluded before both correlation and PCA. Typical thresholds: 0% (Off) — no filter; 5–10% — removes near-constant variables in biological datasets (recommended starting point); >15% — aggressive, keeps only highly variable features. Note: CV is undefined for variables with a mean near zero — these are always kept.

The p-value tests the null hypothesis that there is no real correlation in the underlying population (r = 0). Correlated Pairs & Variable Reduction
Why filter variables before PCA?
When several variables capture the same underlying signal, they carry near-identical information. Because PCA weights every variable equally, a cluster of five highly correlated measurements casts five votes for the same direction — inflating that component's explained variance and potentially obscuring other, independent sources of variation. Removing the most redundant variables beforehand gives each independent signal a fairer contribution to the components. That said, filtering is not always necessary: PCA can accept all variables as-is, decomposing the full correlation structure into uncorrelated components without discarding any information. This is a valid approach when there is no domain reason to prefer one correlated variable over another — though components will then be linear combinations of all variables, so individual variable names lose direct meaning in the biplot. One clear exception: if two variables are literally the same measurement in different units (e.g., weight in kg vs lb), removing one before PCA is always cleaner.
Filtering Strategy

How clusters are formed — Two variables are linked if |r| exceeds the threshold above. Clusters are connected components: if A↔B and B↔C both pass the threshold, all three fall into one cluster even if A↔C does not (B acts as a bridge). Each variable belongs to at most one cluster; uncorrelated variables are not listed. Raise the threshold for stricter clusters; lower it for broader ones.

Hub Score is the sum of |r| with all other cluster members — a higher score means the variable is more redundant and is usually the best candidate for removal. Variables are listed highest hub score first. Uncheck any variable to exclude it from PCA; all are kept by default.
